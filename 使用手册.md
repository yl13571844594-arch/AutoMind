## 1. 产品简介

**AutoMind 是一个"会自己动手干活"的 AI 智能体（Agent）平台。**

普通 AI 聊天工具只能"说"；AutoMind 除了聊天，还能**真正在你的电脑上执行操作**：
创建文件、修改代码、运行命令、抓取网页、生成报告——并且在做每一件事之前，
先自己**规划步骤**，做完后自己**检查结果**，出错时自己**纠正重试**。

### 它能帮你做什么？

| 你想做的事 | AutoMind 怎么帮你 |
|-----------|------------------|
| 问问题、写文案、翻译 | 💬 对话模式，像聊天软件一样直接问 |
| 「帮我建一个 Python 项目」 | ⚙️ 工作模式：自动建目录、写配置、装依赖 |
| 「修复 main.py 里的 bug」 | 💻 编程模式：读代码 → 分析 → 改代码 → 验证 |
| 「写一份行业调研报告」 | 🤝 协同模式：研究员+编写员+审阅员多角色协作 |
| 「持续优化这段代码直到测试全过」 | 🔁 循环模式：反复"执行→检查→改进"直到达标 |
| 「每天早上 9 点汇总昨日日志」 | ⏰ 定时任务：后台自动执行并记录结果 |

### 核心特点

- **本地运行、数据不上传**：所有配置、聊天记录、API Key 都存在你自己的电脑上。
- **安全可控**：每次工具调用都经过风险评估与审批门控，高危操作必须你亲自批准。
- **模型自由**：支持 OpenAI / Claude / DeepSeek / Kimi / 百炼(通义) / 智谱 / 豆包 /
  Gemini / Grok / 本地 Ollama，以及任意 OpenAI 兼容中转代理。
- **可扩展**：技能（Skill）、MCP 服务器、插件（Plugin）三套扩展体系。

## 2. 快速上手（零基础五分钟）

### 第 1 步：安装

前提：电脑已安装 **Python 3.11 或更高版本**（命令行输入 `python --version` 可查）。

在软件目录打开命令行（地址栏输入 `cmd` 回车），执行：

```bash
pip install -e ".[web]"
```

> 想使用 Claude / Gemini 官方 SDK 等全部后端，改用 `pip install -e ".[full]"`。

### 第 2 步：启动

三种方式任选其一：

| 方式 | 操作 | 适合 |
|------|------|------|
| **Windows 一键启动** | 双击目录中的 `launch.bat`，选 `[2]` 自动打开浏览器 | 最简单 |
| **命令行** | `python -m automind.server --port 8765`，然后浏览器打开 `http://localhost:8765` | 通用 |
| **Docker** | `docker compose up --build`，访问 `http://localhost:8765` | 服务器部署 |

### 第 3 步：配置模型（只需一次）

首次打开界面，点击右上角 **「🔑 API Keys」**：

1. 找到你有账号的提供商（比如 DeepSeek），把 API Key 粘贴进去，点保存。
2. 点 **「🔌 测试连接」**，看到绿色通过即配置成功。
3. 顶部的模型徽标会变成绿色「已连接」。

> 没有任何 API Key？两个零成本方案：
> - **本地 Ollama**：安装 [Ollama](https://ollama.com) 并 `ollama run llama3.2`，无需 Key。
> - **中转代理**：在 API Keys 面板底部「自定义 OpenAI 标准接口」填入代理的
>   `api_base`、模型名和 Key，可对接任何 OpenAI 兼容服务。

### 第 4 步：执行第一个任务

1. 顶部模式栏点 **「⚙️ 工作」**。
2. 输入框输入：`在当前目录创建 hello.txt，内容为 Hello AutoMind`，回车。
3. 观察：聊天区出现 **「⚙️ 执行过程」面板**，实时显示 计划生成 → 逐步执行 → 完成；
   右侧「📋 执行计划」同步刷新每一步状态（○ 待执行 → ◐ 执行中 → ✓ 完成）。
4. 打开项目目录，`hello.txt` 已经真实创建。

**恭喜，你已经掌握了 AutoMind 的基本用法。** 下面的章节帮你把每个功能用到极致。

| 入口 | 功能 |
|------|------|
| 💬 对话工作台 | 主聊天区（默认视图） |
| 📋 计划视图 | 查看当前/最近任务的分层计划树 |
| 🔧 工具面板 | 管理 工具/技能/MCP/插件 四类能力 |
| 📊 统计分析 | 基础统计（社区版）+ 高级仪表盘：命中率圆环、趋势图、Token 效率（专业版） |

### 顶部栏（控制）

- **模式切换（分组式）**：默认只显示 3 个**主模式**——💬 对话 / ⚙️ 工作 / 💻 编程，
  界面清爽、新手不慌；主模式右侧有一条分隔线和「**⋯ 高级**」按钮，点开后展开
  🤝 协同 / 🔁 循环两个**高级模式**（老用户全火力）。选中某个高级模式时会自动展开，
  确保当前模式始终可见。每个模式的**会话内容互相独立**，切换不会丢失各自的聊天记录。
- **审批下拉**：🙋询问 / ⚡自动 / ✅全批准（详见第 5 章）。
- **状态徽标**：`●已连接`（WebSocket 状态）、`deepseek/xxx`（当前模型，绿=就绪 黄=异常 红=未配置）、
  `📁项目名`（当前项目目录，点击可更换）。
- **📖 手册**：在新标签页打开内置使用手册（即本文档，服务地址 `/manual`），离线可用。
- **🔄 新会话**：清空当前模式的对话（其他模式不受影响）。

### 输入区

- `Enter` 发送，`Shift+Enter` 换行。
- **📎 图片**：上传图片给视觉模型看图问答（对话模式）。
- **🎤 语音**：浏览器语音转文字输入。
- **■ 停止**：中断正在执行的任务（快捷键 `Ctrl+.`）。

### 右侧面板（观测）

任务执行时这里是"仪表盘"：实时步骤数/回溯数/Token/耗时、Token 累计用量、
分层计划树实况、项目中 HTML 文件的一键预览、最近工具调用的审计摘要。

### 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` / `Shift+Enter` | 发送 / 换行 |
| `Esc` | 关闭所有弹窗 |
| `Ctrl+.` | 中断当前任务 |
| `Ctrl+L` | 新会话（清空当前模式对话） |

## 4. 五种交互模式深度解读

### 💬 对话模式 — "问答专家"

**原理**：纯多轮对话，不调用任何工具，直接与大模型交互，响应最快。
支持图片输入（需要视觉模型，如 gpt-4o / claude / qwen-vl）。

**适合场景**：
- 知识问答、概念解释、头脑风暴
- 文案撰写、翻译、润色
- 上传截图/图表提问（「这张报错截图是什么问题？」）

**不适合**：需要动手操作文件/命令的任务（它没有"手"，请切工作或编程模式）。

**小技巧**：对话历史按浏览器隔离持久化，刷新页面不丢；「🔄 新会话」可随时清空重来。

---

### ⚙️ 工作模式 — "项目经理 + 执行团队"

**原理**：Plan-and-Execute 架构。收到任务后：
1. **分层规划**：把大任务拆成目标树（比如「建博客项目」→ 建目录 → 写配置 → 初始化 git → 写首页）；
2. **依赖排序**：自动分析步骤间的依赖（先建目录才能写文件），生成正确执行顺序；
3. **逐步执行**：每一步调用相应工具（终端/文件/浏览器），执行前过权限门控；
4. **符号验证**：每步完成后校验预期效果是否达成；
5. **自我纠错与回溯**：某步失败时让模型分析错误并生成修正方案重新执行，仍不行则回溯调整计划。

**适合场景**：
- 「初始化一个 FastAPI 项目，带健康检查接口和 Dockerfile」
- 「把这个目录下所有 CSV 合并成一个 Excel 并生成汇总图表」
- 「下载 XX 页面的数据并整理成 Markdown 表格」
- 一切**多步骤、有先后依赖**的自动化任务

**观察窗口**：执行时聊天区的「⚙️ 执行过程」轨迹卡片（紫色=计划、绿色=工具调用、黄色=回溯）
和右栏「📋 执行计划」树是理解它"在想什么"的最佳窗口。

**建议**：任务描述越具体，规划质量越高。给出目标目录、技术栈、验收标准会显著提升成功率。

---

### 💻 编程模式 — "结对程序员"

**原理**：ReAct（思考→行动→观察）循环。模型每轮先输出**思考**，
再决定调用什么工具（读文件/改代码/跑命令），观察结果后进入下一轮，
直到判断任务完成。相比工作模式，它**不预先定死计划**，边做边调整，更适合探索性任务。

**适合场景**：
- 「读一下 src/main.py，找出为什么启动报错并修好」
- 「给 utils.py 里的函数补全类型注解和 docstring」
- 「写单元测试覆盖 order 模块，然后跑通」
- 重构、调试、代码审查等**需要反复看代码-改代码-验证**的工作

**执行可视化**：思考（蓝色卡片）与工具调用（绿色卡片）实时流式展示，
你能看到它每一步"为什么这么做"。

**建议**：先在「⚙ 设置」中把**项目目录**指到你的代码仓库根目录，
它读写文件都以此为根（也受路径防护限制，不会越界操作）。

---

### 🤝 协同模式 — "多角色专家组"（专业版）

> 🔒 本模式为**专业版**功能：社区版中按钮显示 🔒，点击会提示升级；
> 安装 `automind-pro` 并配置许可证后自动解锁。

**原理**：多智能体编排。系统把任务分派给多个角色（规划者/研究员/编程者/审阅者），
各自基于共享白板完成子任务，最后综合产出。角色间互相"看得到"彼此的成果。

**适合场景**：
- 「调研三个开源向量数据库的优劣并给出选型建议」（研究员搜集→审阅者把关）
- 「设计一个签到系统：给出方案、代码骨架和风险评估」（跨角色长任务）
- 需要**多视角**（技术+产品+风险）综合判断的复杂问题

**观察窗口**：聊天区会显示「🤝 多智能体协同」进度面板，逐角色展示分工与产出。

---

### 🔁 循环模式 — "不达目标不罢休"（专业版）

> 🔒 本模式为**专业版**功能，解锁方式同上。
> 社区版的工作/编程模式仍内置「自主任务闭环」（§8.0），基础的自动验收与修复不受影响。

**原理**：Loop Engineering 自主闭环：执行 → 用模型判断"目标达成了吗" → 未达成则带着
反馈继续下一轮。内置**四重停止条件**防止无限循环：
1. **语义完成**：模型判定任务已达标；
2. **输出收敛**：连续两轮输出高度相似（相似度>0.95），说明已无改进空间；
3. **连续空转**：两轮没有执行任何实际操作；
4. **最大轮数**：兜底上限。

**适合场景**：
- 「持续优化这个函数的性能，直到基准测试 <100ms」
- 「反复运行测试并修复失败项，直到全部通过」
- 「把这篇文章改到 500 字以内且保留所有要点」
- 一切**有明确验收标准、需要迭代逼近**的任务

**观察窗口**：「🔁 循环工程」面板显示每轮的动作与判定结论及停止原因
（完成/收敛/空转/上限）。可随时点 ■ 中断。

**建议**：任务里**写清楚验收标准**（"直到测试全过"、"直到没有 lint 错误"），
判定器才有明确依据。

```

## 5. 工具审批与安全体系

AutoMind 的一大特色是**每次工具调用都被安全体系拦截评估**，你可以精确控制"它能做什么"。

### 5.1 三种审批模式（顶部下拉切换）

| 模式 | 行为 | 适合场景 |
|------|------|---------|
| 🙋 **询问** | 除只读操作（读文件、查目录）外，**每次**工具调用都弹窗请你批准 | 首次使用、操作重要项目、想完全掌控 |
| ⚡ **自动**（默认） | 低风险自动放行；风险分 ≥60 的敏感操作和所有高危操作弹窗确认 | 日常使用的最佳平衡 |
| ✅ **全批准** | 跳过所有审批，完全自主运行 | 沙箱环境跑批量任务；**慎用** |

弹窗出现时会显示：工具名、参数、风险等级与原因，你选择「批准」或「拒绝」。
拒绝后模型会收到反馈并尝试其他途径或放弃该步。

### 5.2 权限分级（系统内部如何评估）

每个工具自带权限等级 + 每次调用动态评估风险分（0-100）：

- **safe（安全）**：读文件、列目录、`git status` 等只读操作 → 直接放行；
- **sensitive（敏感）**：写文件、装依赖、`git commit` 等 → 按审批模式处理；
- **dangerous（高危）**：`rm -rf`、`sudo`、强制推送、格式化等 → 必须人工批准。

另有多重硬防护独立于审批模式**始终生效**：
- **命令注入拦截**：`rm -rf /`、fork bomb、写磁盘设备等灾难性命令直接拒绝执行；
- **路径防护**：文件读写被限定在项目目录内，`../` 越界访问直接拒绝；
- **Python 沙箱**：代码执行工具只允许白名单模块（math/json/re 等），禁止 os/subprocess。

### 5.3 审计日志（🛡️ 安全审计）

每次工具调用（无论放行或拒绝）都被记录：时间、工具、参数摘要、决策、风险分、原因。
侧边栏「🛡️ 安全审计」查看完整记录与汇总（放行/询问/拒绝/高危次数）。
**用途**：事后追溯"它到底做过什么"，尤其在全批准模式跑完任务后审查。

## 7. 工具面板：工具 / 技能 / MCP / 插件

侧边栏「🔧 工具面板」下有四个分段，对应四类能力：

### 7.1 🔧 工具（Agent 的"手"）

内置工具一览：

| 工具 | 等级 | 功能 |
|------|------|------|
| `terminal` | 敏感 | 执行命令行（装依赖、跑脚本、git 操作） |
| `file_read` | 安全 | 读取文件内容 |
| `file_write` | 敏感 | 创建/覆盖文件 |
| `file_edit` | 敏感 | 精确字符串替换编辑（带 diff 与回滚） |
| `python_sandbox` | 敏感 | 沙箱内执行 Python 片段（计算/数据处理） |
| `web_fetch` | 安全 | 抓取网页并提取正文与链接 |
| `browser` | 敏感 | Playwright 浏览器自动化（导航/点击/输入/截图） |

每个工具卡片有**开关**：临时关掉某工具，Agent 执行时就"看不到"它。
**场景**：不想让它上网 → 关掉 `web_fetch` 和 `browser`；只读分析 → 只留 `file_read`。

### 7.2 ✨ 技能（预制的"专家流程"）

技能 = 封装好的多步专业流程，模型可一键调用。内置 6 个：

| 技能 | 功能 | 典型用法 |
|------|------|---------|
| `project_init` | 项目脚手架（目录/pyproject/venv/git） | 「初始化一个新项目」 |
| `code_generator` | 代码生成/**补全**/脚手架，带语法校验与自动修复 | 「生成一个爬虫脚本」 |
| `test_runner` | 发现并运行测试、收集失败 | 「跑一下测试」 |
| `log_analyzer` | 日志级别统计/错误样本/高频模式聚类 | 「分析这个日志文件」 |
| `doc_generator` | 从 Python 源码生成 Markdown API 文档 | 「给 src 目录生成 API 文档」 |
| `dep_audit` | 依赖审计：未固定版本/重复声明/已装版本 | 「检查项目依赖健康度」 |

**导入自定义技能**（两种格式）：
- **SKILL.md 技能包**：文件夹内放一个 `SKILL.md`（YAML 头写 name/description/emoji，
  正文写指令），适合无代码的提示词型技能。「📁 加载目录」批量导入。
- **Python 技能**：继承 `AbstractSkill` 的 `.py` 文件，「📄 导入 .py」。
  开发模板见 `examples/03-skill-development/`。
- 「⬇️ 一键导入桌面 skills 文件夹」可直接吸收桌面的技能库。

### 7.3 🔌 MCP（连接外部工具生态）

MCP（Model Context Protocol）是开放协议，接入后外部服务的工具会
**自动出现在工具列表里**供 Agent 调用（文件系统、数据库、搜索、地图……）。

- **单个添加**：名称 + 传输方式（stdio 本地命令 / sse URL）+ 命令与参数。
- **批量导入**：直接粘贴 Claude Desktop 格式的 `{"mcpServers": {...}}` JSON。
- 需先 `pip install mcp`（面板会提示）。
- 连接状态、每个服务器暴露的工具一目了然，可随时删除。

### 7.4 🧩 插件（生命周期钩子扩展）

插件在任务**开始/结束/解析后/计划后/出错**等时机自动执行你的自定义逻辑
（记日志、发通知、审计上报），**不改动主程序**。

- 插件放在 `~/.automind/plugins/<名称>/`，含 `plugin.json`（元信息）+
  `hooks.py`（提供 `get_hooks() → AgentHooks`）。
- 面板中点「加载/卸载」即时生效；「🔄 重新扫描」发现新插件。
- 插件内部异常会被系统吞掉，**永远不会影响任务主流程**。
- 完整开发示例见 `examples/04-plugin-development/`（task-timer 计时插件）。

## 8. 高级功能

### 8.0 🔄 自主任务闭环（v0.3 新增，默认开启）

**做什么**：工作 / 编程模式的任务完成后，系统自动执行一条"质检流水线"，
不合格就**自动返工**——把"AI 说做完了"变成"验证过真的做完了"。

```
任务执行完成
   │
   ▼
🧪 TDD 测试（编程模式）      项目有 tests/ 时自动跑 pytest
   │                        （执行中每次改 .py 还会即时语法验证）
   ▼
🧐 多 Agent 审查（工作模式）  审阅者角色复核结果，可调用只读工具核实
   │                        （工具共享同一注册表，MCP 工具同样可用）
   ▼
✅ Loop 语义验收             判定"任务是否真正完成"
   │
   ├─ 通过 → 输出末尾附「🔄 自主闭环」摘要
   └─ 未过 → 带着反馈自动修复（最多 2 轮）后再次验收
```

**五个开关**（「⚙ 设置」→ 通用 → 自主任务闭环，默认全开、即时生效）：

| 开关 | 作用 | 关闭的场景 |
|------|------|-----------|
| 🧐 多Agent审查 | 工作模式完成后审阅者复核 | 追求极致速度 / 省 Token |
| ✅ Loop 验收 | 语义验收 + 未过自动修复（≤2 轮） | 简单任务不需要返工 |
| 🧪 TDD 测试 | 编程模式改 .py 即时语法验证 + 收尾跑 pytest | 项目测试极慢时 |
| ⚡ 并行执行 | 计划中互不依赖的步骤并发执行 | 步骤间有隐式资源冲突时 |
| 📦 子任务缓存 | 同任务内相同只读调用（读文件/抓网页）复用结果 | 需要每次强制重读时 |

**编程模式的 TDD 内环**：每次 `file_write`/`file_edit` 写入 `.py` 文件后，
系统立即做语法检查并把 `syntax_check: OK / FAILED` 注入模型的观察结果——
有语法错误它下一步就会看到并立刻修复，形成 **编辑 → 验证 → 修复** 的真正 TDD 闭环。
另有 `code_generate` 工具（背后是 code_generator 技能）：语言检测、
语法校验 + 自动修复、`mode='complete'` 代码补全。

**安全边界**：并行只发生在互不依赖的步骤之间；缓存只作用于 SAFE 级只读工具
（写文件、执行命令永远真实执行）；审阅者只能调用只读工具。

### 8.1 ⏰ 定时任务

> 🔒 **专业版**功能：社区版侧边栏入口显示 🔒，打开后为升级说明卡片。

**做什么**：让某个任务按固定周期自动执行（无需开着页面盯着）。

**创建**：侧边栏「⏰ 定时任务」→ 填任务内容 → 选模式（五种均可）→
选间隔（每5分钟/每小时/每6小时/每天）→ 添加。

**典型场景**：
- 每天汇总一次项目日志中的错误（配 `log_analyzer` 技能）
- 每小时抓取某页面数据追加到 CSV
- 每 6 小时跑一遍测试并记录结果

每次执行的结果记入任务历史；可随时暂停/启用/手动触发/删除。重启服务后自动恢复调度。

### 8.2 📊 统计分析（高级仪表盘，专业版）

> 社区版保留**基础统计**（任务总数/成功率/平均耗时/Token 累计/按模式聚合/工具排行）；
> 以下高级仪表盘为**专业版**功能，社区版页面底部会显示升级卡片。

- **四个命中率圆环**：工具命中率 / 计划命中率 / 任务成功率 / 自我修正率 + 综合平均。
- **上下文使用率发光条**：当前对话窗口占模型上下文的百分比，是否触发过压缩。
- **趋势折线**：最近 50 次任务的工具命中率变化。
- **Token 效率**：每 Token 产出的有效字符数；输入/输出分项累计。
- **记忆系统**：长期记忆文档数、知识图谱实体/关系数、短期窗口消息数。

**用途**：评估"这个模型/这种提示词效果如何"、监控 Token 花销、发现频繁回溯的异常任务。

### 8.3 🌐 HTML 预览

Agent 生成的 HTML 文件（报告/图表/小页面）出现在右栏「🌐 HTML 预览」列表，
点击即可在弹窗内直接渲染查看，无需手动找文件。

### 8.4 多模态输入

- **📎 图片**：对话模式下上传图片（可多张），配视觉模型即可看图问答、读截图、分析图表。
- **🎤 语音**：点击后说话，浏览器实时转文字进输入框（基于 Web Speech API，需浏览器支持）。

### 8.5 多用户会话隔离

每个浏览器持有独立的会话 ID：A 电脑与 B 电脑打开同一服务，聊天记录互不可见、
互不清除。团队共用一台服务器部署时无串扰。（社区版即支持）

> 👥 **执行态隔离（会话级 Agent 池）为企业版功能**：并发执行工作/编程任务时
> 每个会话拥有独立执行体（独立计划/ReAct 状态/审批回调），互不干扰；
> 激活企业版后默认开启，`AUTOMIND_SESSION_POOL=0` 可关闭。

### 8.6 输出便捷操作

- 每条 AI 消息悬浮 **「⧉ 复制」**（复制整条 Markdown 原文）。
- 每个代码块带语言标签头和独立 **「⧉ 复制」** 按钮。

## 9. 命令行与 REPL

# 单次任务（默认工作模式）

automind --list-tools       # 可用工具

automind --list-providers   # 支持的提供商
```

### Rich REPL（交互式终端）

不带参数运行 `automind` 进入彩色交互终端：

```
+---------------- REPL ----------------+
| AutoMind v0.2.0                      |
| 模式: plan_and_execute · 模型: ...    |
+--------------------------------------+
automind> /help
```

REPL 内命令：

| 命令 | 功能 |
|------|------|
| `/help` | 帮助 |
| `/mode react` 或 `/mode plan_and_execute` | 切换执行模式 |
| `/tools` / `/skills` | 表格列出工具/技能 |
| `/stats` | 本次会话任务数与 Token 统计 |
| `/clear` | 清屏 |
| `/exit` | 退出（显示 Token 总量摘要） |
| 其他任何输入 | 作为任务执行，结果以 Markdown 面板渲染 |

## 11. 常见问题 FAQ

**Q：顶部模型徽标一直是红色「未配置」？**
A：还没配 API Key。点「🔑 API Keys」为当前提供商填 Key 并「测试连接」。

**Q：徽标黄色带 ⚠？**
A：有 Key 但初始化失败。常见原因：Key 失效、`api_base` 写错、网络不通。
「测试连接」会告诉你卡在哪个阶段。

**Q：任务卡住不动 / 想中断？**
A：点输入区 ■ 或按 `Ctrl+.`。循环模式还会在收敛/空转时自动停止。

**Q：为什么它说「文件不存在」？**
A：检查右上角 📁 项目目录是否指向了正确位置——所有相对路径都以它为根，
且出于安全不允许访问目录之外的文件。

**Q：担心它乱删文件？**
A：三层保险：① 审批切到「🙋 询问」逐一批准；② 高危命令（rm -rf 等）被硬拦截；
③ 事后在「🛡️ 安全审计」核查每一次调用。

**Q：换台电脑打开，聊天记录不见了？**
A：正常。会话按浏览器隔离（多用户设计）。记录都在服务器本地 `.automind/chats/` 下。

**Q：Token 花得快？**
A：① 在「🖥 模型」给对话模式配便宜模型；② 「📊 统计分析」看 Token 效率定位大户任务；
③ 任务描述精准、避免让它反复试错。

**Q：协同/循环按钮带 🔒 点不动、定时任务打开是升级卡片？**
A：这些是**专业版**功能，当前运行的是社区版。安装商业扩展包 `automind-pro`
并配置许可证（环境变量 `AUTOMIND_LICENSE`，或项目根目录 `.automind_license` 文件）
后重启服务即自动解锁；左上角徽标会从「社区版」变为「专业版/企业版」。
许可证无效或过期会自动降级社区版，不影响其他功能。

**Q：社区版功能会被砍吗？**
A：不会。对话/工作/编程、全部工具/技能/MCP、安全体系（鉴权/限流/脱敏/审计）
承诺保持开源免费；详见 `docs/EDITIONS.md`。

**Q：想让它用我公司内部的工具？**
A：三选一：写 Python 技能（最简单，见 examples/03）；架 MCP 服务器（标准协议，生态互通）；
写插件（适合旁路逻辑如通知/审计）。
